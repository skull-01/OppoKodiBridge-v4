01

The core idea

v4's defining move is no-blip handoff via Kodi's playercorefactory. Instead of a service that lets Kodi start a disc and then yanks it back (the v2 "blip"), the service writes a playercorefactory.xml that routes .iso / BDMV / VIDEO_TS content to an external player before Kodi's own player ever opens the file.

That one decision splits the add-on into two processes with a JSON file as the bridge between them.

in-Kodi · long-lived

Service process

service.py → service_cec.main()

The only code with xbmcaddon access. Publishes runtime_config.json, installs the playercorefactory.xml, and hosts the two always-on features (volume takeover, remote passthrough). Never intercepts playback.

external · per disc

External player process

pcf_player.py → orchestrator.run()

Spawned by Kodi for each disc, runs outside Kodi, reads the published runtime_config.json. The entire handoff lifecycle — detect, switch, play, watch, reclaim — lives here.

A third, separate companion add-on — script.cecreclaim — runs inside Kodi purely to re-announce Kodi's own HDMI-CEC active source on stop, so the TV returns to Kodi.
02

Component topology

The Kodi host holds the software; the four physical devices sit outside it. Every link is labelled with its protocol — HTTP :436 (OPPO app API), SSH (to the Pi IR blaster), JSON-RPC :8080 (localhost, into Kodi), plus HDMI-CEC and IR.

03

Runtime flows

Disc-handoff lifecycle

orchestrator.run is a clean linear flow — single-shot and event-tied.

  1. Kodi plays a disc — playercorefactory routes it to pcf_player.py, which loads runtime_config.json.
  2. detector.is_handoff_target — accept only .iso / BDMV / VIDEO_TS; anything else goes back to Kodi.
  3. tvswitch.to_oppo() — switch the TV to the OPPO input (CEC grab or IR blast — see below).
  4. handoff.play() — drive the OPPO over :436: wake → init → NFS login/mount → play. Mount and play are hard gates (abort-before-play on an unconfirmed mount).
  5. monitor.watch_playback() — poll /getglobalinfo until idle; one ISO auto-heal re-issue, bounded so it always returns.
  6. tvswitch.to_kodi() — reclaim the TV, in a finally so it runs whether playback succeeded or failed. No standing re-asserter, so a manual input change sticks.

Volume takeover over IR

The M920x's network volume is inert, so the remote's Vol± drive the TV over IR instead: a Kodi keymap remaps the volume keys to NotifyAll; onNotification maps it to an RCA code; a worker thread fires it at the TV through a persistent SSH pipe to the Pi.

Remote passthrough

While a handed-off disc plays, an invisible full-screen dialog captures the remote, resolves each key to an OPPO OREMOTE code, and forwards it over :436 /sendremotekey on a worker thread — so you can drive the Blu-ray menu.

TV-switch strategies

tvswitch.make_switcher() selects an implementation from tv_switch_method, all behind one to_oppo() / to_kodi() contract — each single-shot, non-fatal, returning an honest bool. The default cec reproduces the original inline behaviour exactly; IR transports load lazily.

07

Design principles

testability

Pure cores, off-box testable

detector, config, passthrough, ir_proto, and the volumeir/wizard logic import no xbmc, so the suite runs without hardware. All I/O sits behind thin adapters.

two processes

A JSON file is the bridge

The external player can't call xbmcaddon, so the in-Kodi service dumps resolved settings to runtime_config.json for it to read.

consistency

Single source of truth

The playercorefactory routing rules are derived from the same detector constants the runtime matches on — the XML can't drift from the in-process check.

CEC discipline

Assert once per event

Every TV switch is single-shot; there is no standing re-asserter, so if you change the TV input by hand, your choice stays.

resilience

Best-effort, non-fatal

Config publish, TV switch, give-up STOP, keymap sync, and all IR / volume / passthrough I/O are wrapped so a failure never skips the reclaim or crashes the service.

safety

Default-off for anything new

IR transports, remote passthrough, and volume takeover all ship off; the proven CEC handoff path is unchanged unless you opt in.
